This framework uses pug as default template engine and convert it to html file. It also uses Vue as default JS library. Feel free to change it if you want.


Workflow

Directory structure

The root contains two main directories: assets and build. The assets directory is where all the compiled code for .js and .css goes to. In the root you also have the index.html file, which is where you .pug file is compiled.

The source directory is where you will work properly. Here you have a little structure to be easy to maintain the code more organized.


The /source directory

> The js dir is where you will place your js code. It have other two directories: build and source. The source is where you will place the code itself. Here you can split you code into separate files, which is better to maintain it organized. The build directory is where all code insde the source dir is joined into only one code. It's the previous step before Gulp minify it and place it in the root/assets/js directory.

> The scss directory is where you will place all your .scss stylesheet files. All the structure is explained in the style.scss file.

> The view directory is where you will place your .pug files, which will be compiled into .html files and placed in the root.
